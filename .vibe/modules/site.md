# Module: site
**Role:** Generates a styled static HTML site to browse the registry's content (games grouped by system, with name, description, and jaquette, sticky horizontally-scrolling navigation between systems, click-to-expand full game details including metadata and video) in a web browser.
**Files:** `internal/site/site.go`
**Exports:**
- `site.Generate(reg *registry.Registry, registryFolder string) error` — groups `reg`'s entries by system (sorted alphabetically, games sorted by name within each system for deterministic output) and writes `<registryFolder>/index.html`, directly at the registry's root. Each system is rendered as an anchored section (`id="<system>"`) linked from a sticky navigation bar (scrolling horizontally, rather than wrapping, when many systems are configured), and each game as a styled card with a clamped (truncated) description; a game without a jaquette, or whose referenced jaquette/video file no longer exists on disk in the registry folder, renders a placeholder card art instead of a broken `<img>`/`<video>`. Each card links to a per-game detail overlay showing its full, untruncated description plus rating (as stars), release year, developer, publisher, genre, players, and gameplay video (when available and present on disk), shown/hidden with a pure-CSS `:target` technique (no JavaScript) — see [`decisions/010`](../decisions/010-css-only-detail-modal-via-target.md). The overlay's close controls link to a non-existent anchor so closing it never scrolls the page — see [`decisions/011`](../decisions/011-void-anchor-to-close-modal-without-scrolling.md). Every system section ends with a link back to the top of the page. An empty registry still produces a valid page, with a "No games in the registry yet." message. Media (jaquette, video) references are resolved relative to the generated page as `<system>/<path>`, with each path segment percent-encoded so file names containing spaces, brackets, or parentheses still produce valid links, mirroring where `registry.ImportFromRomsFolder` copies media under the registry folder. The page's CSS is embedded inline in a `<style>` block (no external stylesheet), including a responsive rule so the layout stays readable on small screens.

**Depends on:** [`modules/registry.md`](registry.md), [`modules/gamelist.md`](gamelist.md)

**Architecture note:** Added for backlog item 007. Rendering uses Go's `html/template` (not `text/template`) so game names/descriptions are auto-escaped, since they originate from user-editable `gamelist.xml` files. Regeneration is not exposed as its own CLI command — see [`modules/cli.md`](cli.md) and [`decisions/006`](../decisions/006-auto-regenerate-html-site-on-update.md): it is instead triggered automatically at the end of the `update` command, the only command that actually mutates the registry. The generated page moved from a `site/` subfolder to the registry root for backlog item 008 — see [`decisions/008`](../decisions/008-move-consultation-site-to-registry-root.md); a `site/` folder left over from a previous version is never auto-deleted, by explicit product decision. The navigation bar is sticky (stays visible while scrolling) rather than a one-time summary block, for backlog item 010 — see [`decisions/009`](../decisions/009-sticky-navigation-bar-on-consultation-site.md). The generated page stays JavaScript-free by using a CSS-only `:target` show/hide technique for the per-game detail overlay — see [`decisions/010`](../decisions/010-css-only-detail-modal-via-target.md) and [`decisions/011`](../decisions/011-void-anchor-to-close-modal-without-scrolling.md). Media paths are built with a dedicated helper that percent-encodes each path segment (`net/url`'s `PathEscape`) instead of relying on `html/template`'s own URL context escaping, because the latter deliberately leaves characters like `[` and `]` unescaped in URL attributes, which some HTTP servers mishandle. `groupBySystem` checks each referenced jaquette/video file's presence on disk (`os.Stat` under the registry folder) before building its (percent-encoded) URL, since a game's metadata can reference a file the registry no longer actually has; a missing file is treated the same as no file at all (placeholder card art, no video player).
